SRP : 단일 책임 원칙

중 SRP는 싱글톤 패턴, 첵임 연쇄 패턴을 사용한다. 

싱글톤 패턴 : 클래스의 인스턴스가 하나만 있도록 하면서 이 인스턴스에 대한 전역 접근 지점을 제공 하는 생성 디자인 패턴
하나만 클론이 되게 하다

보안과 효율을 위해서 사용됨
"슈퍼 인스턴스"



세 분류의 영역 
프레임워크에서 클래스 -> 메서드 -> 에어리어, 필드 <- 컨텍스트

전역 : 컨텍스트
로컬 : 에어리어
인스턴스 : 필드

한 번에 두 가지의 문제를 해결함으로써 단일 책임 원칙을 위반

책임 연쇄 패턴 : 헨들러의 체인(사슬)을 따라 요청을 전달할 수 있게 해주는 행동 디자인 패턴. 
각 헨들러는 요청을 받으면 

커맨드 패턴 : 요청(Request)을 요청에 대한 모든 정보가 포함된 독립 실행형 객체로 변환(Schema)하는 행동 디자인패턴
